# Observations

Observations are string contents attached to entities, used for context and memory.

## Stored Content

- Agent execution logs: `input`, `output`, `sync`
- User preferences and metadata
- Workflow state and results
- Deploy logs and status

## Examples

- `Memory MCP enables cross-session context`
- `Sequential Thinking improves agent decisions`
- `GitHub MCP automates code management`
- `sync: {"agentType":"research","action":"search",...}`
