Feat: Implement enhanced authentication and secure API access

This commit introduces several security and robustness enhancements across the AgentFlow Pro application:

- **Enhanced API Authentication**: All critical API routes (usage, billing, workflow execution, workflow management, memory entities, memory graph, user API keys, Stripe checkout) now require a valid user session. Unauthorized access attempts will result in a 401 Unauthorized response.
- **User-Scoped Data Access**: Workflow and memory operations are now strictly user-scoped, ensuring users can only access and manage their own data.
- **Strict Orchestrator Mode**: The orchestrator factory now supports a strict mode, preventing agent registration if required API keys are missing. This improves reliability by ensuring agents only run when fully configured.
- **API Key Management Improvements**: Users can now delete their stored API keys by sending an empty string for a given provider, improving key management flexibility.
- **Secure Stripe Checkout Origin**: Stripe checkout redirects now use `process.env.VERCEL_URL` for the origin URL, enhancing security and reliability in Vercel deployments.
- **Refactored User ID Retrieval**: Standardized user ID retrieval across API routes using `getUserId` from `@/lib/auth-users`, promoting consistency and security.
- **Improved Error Handling**: Added `console.error` logging in API routes and workflow services for better server-side error visibility.

These changes significantly strengthen the application's security posture, enforce data isolation, and provide a more robust operational environment.
